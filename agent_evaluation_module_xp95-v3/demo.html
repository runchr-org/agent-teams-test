Each WORD in the response is wrapped in a colored span that cycles through 8 hues for tiktoken-viz visual separation. The COLOR FAMILY signals diction-level diff with Agent B: • warm (coral / peach / gold / salmon) — words unique to this agent, NOT used by Agent B. Drift the harness suppressed when injected. • muted greyscale — words shared with Agent B. Any 3+ consecutive content words appearing in both responses tint as shared. The "baseline model voice" both agents share. At scroll speed: more warm spans = more raw-side drift; more grey = both agents converged.

Results Overview?
Three columns of blind-judge results. • Latest run · dimension scores: per-dimension 0-10 scores. R = raw, D = dynamic, A = adaptive. ★ marks the top score; the coloured number is the delta vs Raw. • Mean per dimension: averages across all runs this session, with ±stddev once there are 2+ runs so a small gap reads as noise rather than a win. • Cost · latest run: latency, total tokens, answer length (chars), reasoning tokens, and real USD cost (when the provider reports it). Answer chars is a length-bias check: LLM judges tend to prefer longer answers, so if a harness condition wins while also being much longer, treat the win with caution. Harness conditions usually cost more because the scaffold is injected. The strip under each answer is the per-token confidence ribbon: solid = the model committed; pale = it was uncertain (from logprobs).

Configuration
Agent B uses the selected harness domain (dynamic); Agent C uses its adaptive variant. Same prompt and model for all three; only the scaffold differs. The three judges should be different vendors so they do not share blind spots. Seed pins sampling for reproducible runs (blank = provider default). Pin provider forces every call to one OpenRouter upstream host (comma-separated for fallback order) so all three agents run on the identical model build; blank lets OpenRouter load-balance, which can put each agent on a different host. Max answer tokens caps each response (default 1024); raise it if harness answers get truncated, since a cut-off answer scores worse for the wrong reason. Provider keys are localStorage-only; the Ejentum key is prefilled.
